# M5-6: サーバーログ＆監視

読むファイル: GameModeManager.cs, HitboxSystem.cs, GameConfig.cs
変更ファイル: 新規 Scripts/Server/ServerLogger.cs, GameModeManager.cs, HitboxSystem.cs, GameConfig.cs

サーバー側のイベントログと監視システムを実装する。

## 実装内容

1. 新規 Scripts/Server/ServerLogger.cs：
   - MonoBehaviour（サーバー専用、シングルトン）
   - ログレベル: Info / Warning / Error
   - タイムスタンプ + ログレベル + メッセージのフォーマット
   - ファイル出力（ローカル、日付ローテーション）
   - 将来: Firestore への非同期書き込み（バッチ送信）

2. ログ対象イベント：

| イベント | レベル | 内容 |
|---------|--------|------|
| プレイヤー接続 | Info | clientId, UID, IP |
| プレイヤー切断 | Info | clientId, 理由 |
| 試合開始 | Info | マップ、チーム構成 |
| 撃破 | Info | killer, victim, 使用技, ダメージ量 |
| 試合終了 | Info | スコア、勝敗、MVP |
| チート検出 | Warning | clientId, 違反種別, 詳細 |
| チートキック | Warning | clientId, 累積違反数 |
| 例外 | Error | スタックトレース |

3. HitboxSystem.cs 修正：
   - ヒット確定時にServerLogger.LogKill()を呼ぶ（撃破時のみ）

4. GameModeManager.cs 修正：
   - 試合開始/終了時にServerLoggerに通知
   - 試合結果サマリー（チームスコア、個人成績）

5. GameConfig.cs に追加：
   - LOG_FILE_PATH = "server_logs/"
   - LOG_MAX_FILE_SIZE_MB = 10
   - LOG_RETENTION_DAYS = 7
