# M6-1: キャラクターモデル＆アニメーション

読むファイル: PlayerMovement.cs, ComboSystem.cs, CharacterStateMachine.cs, ReactionSystem.cs
変更ファイル: PlayerMovement.cs, ComboSystem.cs, 新規 Scripts/Character/AnimationController.cs

箱人間からキャラクターモデルに置き換える。

1. 新規 Scripts/Character/AnimationController.cs：
   - NetworkBehaviour（アニメーション同期）
   - Animator コンポーネントとの連携
   - CharacterStateMachine のステート変更イベントに応じてアニメーション再生
   - ステートごとのアニメーションマッピング:
     - Idle, Move, Jump, Guard, Attack(N1-N6), Charge(C1-C6)
     - DashAttack, Musou, Hitstun, Launch, Down系, Getup, Dead
   - NetworkVariable<int> でアニメーション状態を同期（他プレイヤー表示用）
   - 攻撃モーションのコンボウィンドウとアニメーション同期

2. モデル要件：
   - 人型モデル（アセットストアから調達 or 仮モデル）
   - 武器種ごとに異なるアニメーションセット（将来。仮は共通）
   - Humanoid Rig（Mecanim互換）

3. PlayerMovement.cs / ComboSystem.cs：
   - 攻撃時間をアニメーション長に合わせて調整する仕組み
   - AnimationController からモーション長を取得

4. Unity設定（手動）：
   - キャラクターモデルをインポート
   - Animator Controller 作成
   - NetworkPlayer Prefab のメッシュを差し替え
