# M5-4: ロビー＆マッチメイキング

読むファイル: RelayManager.cs, AuthManager.cs, TeamManager.cs, GameConfig.cs
変更ファイル: GameConfig.cs, 新規 Scripts/Netcode/LobbyManager.cs, 新規 Scripts/UI/LobbyHUD.cs

Unity Lobby Serviceを使ったロビー＆マッチメイキングを実装する。

1. 新規 Scripts/Netcode/LobbyManager.cs：
   - MonoBehaviour（シングルトン）
   - Unity Lobby SDK初期化
   - ロビー作成（ホスト）/ ロビー検索・参加（クライアント）
   - クイックマッチ: 空きロビーを自動検索して参加
   - ロビーデータ: マップ、ルール設定
   - プレイヤー準備状態の同期
   - 全員準備完了 → Relay接続 → ゲーム開始
   - ハートビート（ロビー維持用の定期送信）

2. 新規 Scripts/UI/LobbyHUD.cs：
   - MonoBehaviour（クライアント専用UI）
   - ロビー画面: プレイヤーリスト、チーム表示、準備ボタン
   - クイックマッチボタン
   - ルーム作成ボタン（ホスト用）
   - JoinCode入力欄（手動参加用）

3. GameConfig.cs に追加：
   - LOBBY_HEARTBEAT_INTERVAL = 15f
   - LOBBY_POLL_INTERVAL = 2f
   - LOBBY_MAX_PLAYERS = 8

4. Unity設定（手動）：
   - Unity Dashboard で Lobby Service を有効化
   - com.unity.services.lobby パッケージ追加
